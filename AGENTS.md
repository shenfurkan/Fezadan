## Workflow

- Start with `git status --short`; the worktree may contain user changes. Do not revert files you did not intentionally change.
- Prefer the smallest correct change. Keep existing route names, field names, IDs, and JS selectors unless the task requires changing them.

## Commands

```bash
composer install
docker-compose up -d
docker-compose up -d --build
npm run dev
npm run build
npm test
npm run test:e2e
npm run preflight
php scripts/migrate-db.php
php tests/run.php
php cron/generate-sitemap.php --force
php scripts/smoke-test.php
python scripts/check-site.py
```

- `npm test` is `npm run test:e2e`; Playwright runs `tests/e2e` with one Chromium worker and writes `playwright-report/` plus `test-results/e2e-junit-results.xml`.
- `npm run preflight` runs, in order: Docker `php scripts/migrate-db.php`, Docker `php tests/run.php`, `npm run build`, then `npx playwright test`.
- `npm run dev` watches `public_html/assets/css/input.css` into `public_html/assets/css/style.css`; `npm run build` writes the minified production CSS.
- Docker exposes PHP/Apache at `localhost:8080`, MariaDB 10.3 at `localhost:3306`, phpMyAdmin at `localhost:8000`, and LiteCaptcha at `localhost:8089`.
- `docker-compose.yml` mounts LiteCaptcha from `D:\litecaptcha\public`; local captcha tests fail if that sibling checkout is missing.
- `php tests/run.php` is a custom regression runner, currently 56 tests; it uses DB env from Docker or `.env` and allows R2 fallback in testing.
- `scripts/check-site.py` expects `localhost:8080`, `notlar.localhost:8080`, the local admin account `antigravity` / `Antigravity2026!`, and Firefox Developer Edition.
- `scripts/integration_tests.php` targets `http://127.0.0.1` without port 8080, so it is not the default Docker-host check unless port 80 routes to this app.

## Runtime Shape

- This is a custom PHP MVC app, not Laravel/Symfony. `public_html/` is the Apache document root; `public_html/index.php` loads config, CSP/security headers, core classes, Composer autoload, then `app/Core/App.php`.
- `App::ROUTABLE_ACTIONS` is the route allow-list. Adding a controller method is not enough; add it there or the route 404s.
- Main-site URLs require `/tr` or `/en`. `/` redirects with 307 from cookie/Accept-Language; other missing-prefix paths redirect to `/tr/...` except excluded asset/admin paths.
- Admin lives at `/yonetim`; `/admin`, `/panel`, and `/dashboard` intentionally 404.
- `notlar.*` hosts bypass language routing, only allow `GET`/`HEAD`, and route to `NotesController` for PDF archive pages and R2 streaming.
- `/anonymitycheck/` is handled by `public_html/anonymitycheck/index.php`, which rewrites the request host as `anonymitycheck.localhost` before loading the main entrypoint.
- Static pages are resolved by `App::resolveStaticPage()` and must exist under `app/Views/front/`.

## Config And Data

- `app/Config/config.php` has a custom `.env` parser, then falls back to process env. README mentions `.env.example`, but this checkout does not contain one.
- If `APP_SECURITY_SALT` is absent or `change-me`, config generates ignored `app/Config/.security_salt`; never commit it.
- Use `Db::pdo()` for database access; it creates one PDO per request with emulated prepares disabled.
- Schema changes are mostly self-healing in `scripts/migrate-db.php`; run it after DB-affecting changes. Standalone SQL migrations live in `migrations/`.
- Generated/runtime files are ignored: `storage/`, `logs/`, `tmp/`, `scripts/screenshots/`, Playwright outputs, and `public_html/sitemap*.xml`.
- Sitemaps are regenerated by `php cron/generate-sitemap.php --force`; without `--force` the cron exits unless `sys_get_temp_dir()/fezadan-sitemap.dirty` exists.

## Frontend And Assets

- Tailwind is v4 CLI-only: `input.css` uses `@import "tailwindcss"` and `@theme`; there is no `tailwind.config.*`.
- Prefer editing source CSS (`input.css` and imported CSS such as `article-reader.css`) and then running `npm run build`; `style.css` is generated output.
- Executable inline scripts in PHP views need `nonce="<?= CSP_NONCE ?>"`; `public_html/index.php` sends a nonce-based CSP.
- Fonts and frontend libraries are intentionally served locally for privacy. Do not introduce public CDN dependencies unless deliberately preserving an existing admin-editor fallback pattern.

## Uploads And External Services

- Media uploads are designed for Cloudflare R2, not persistent `public_html/uploads/` storage.
- `Upload::saveImageToR2()` validates and re-encodes images, tries WebP, then falls back to original format if WebP tooling is unavailable.
- `Upload::saveOriginalImageToR2()` is used for portfolio images to preserve original format while still validating/re-encoding.
- `R2Storage` throws on missing R2 env outside `APP_ENV=testing`; tests can use the testing fallback.
- Anonymity checks use `storage/anonymity/GeoLite2-ASN.mmdb` when present; `storage/` is ignored, so do not assume the DB exists in a fresh checkout.

## Admin And Security

- `AdminController` is split into traits under `app/Controllers/Admin/`; inspect those before editing admin behavior.
- Mutative admin endpoints should call `requirePost()` and rely on CSRF verification unless they are explicit login/passkey exemptions.
- Admin roles matter: `viewer` is read-only, `editor` manages articles/notes/categories/portfolio, and `superadmin` is required for author/admin management.
- Admin create/edit pages depend on stable element IDs used by `public_html/assets/js/admin-editor.js`; preserve selectors such as `summernote`, `wordImportTrigger`, `manualSlug`, `draftBtn`, and `submitBtn`.
- Notes downloads use LiteCaptcha off localhost and a 3-downloads-per-minute rate limit via `download_rate_limits`.

## Deployment

- `.cpanel.yml` deploys only `app/`, `cron/`, `public_html/`, and Composer files, then runs `/usr/local/bin/php /usr/local/bin/composer install --no-dev --no-interaction --optimize-autoloader`.
- Deploy cleanup removes docs, scripts, migrations, tests, Docker files, root Markdown, and most runtime folders from app/web targets while preserving `storage/anonymity/GeoLite2-ASN.mmdb`.
- `public_html/.htaccess` blocks direct web access to sensitive roots and routes non-files to `index.php?url=...`; update it when adding special public paths.
